# autoresearch-fpl

This is an FPL adaptation of the `autoresearch` loop.

## Scope

You are trying to improve FPL player selection for the next gameweek.

The fixed evaluation harness is:

- `run.mjs` — data loading, feature engineering, backtest, and live report generation. Do not edit this file unless the harness itself is broken.

The editable file is:

- `weights.json` — common weights, position-specific weights, filters, and evaluation pick counts.

## Commands

Run these from the `FPL/` repo root:

```bash
node autoresearch-fpl/run.mjs backtest
node autoresearch-fpl/run.mjs report
```

## Objective

Maximize the backtest quality from `node autoresearch-fpl/run.mjs backtest`.

Primary metric:

- `overall_avg_points` — average actual next-gameweek points scored by the selected top-ranked picks across completed gameweeks.

Secondary metrics:

- position-level average points
- overall hit rate (share of selected players scoring at least 6 points)

## Rules

- Prefer editing only `weights.json`.
- Keep the system simple.
- If a weight change improves backtest output, keep it.
- If it makes backtest worse, revert it.
- After a promising change, run `report` and inspect the new shortlist.

## Current setup

- Historical snapshots come from the `FPL-Elo-Insights` 2025-2026 dataset.
- Live picks come from the official FPL API.
- Recent-history features are computed over the last few completed gameweeks.
- Double gameweeks are rewarded through fixture count and total fixture ease.
